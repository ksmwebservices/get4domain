# Payment Policy — Get4Domain v1.0

## Standard Payment Flow

1. Agreement signed
2. 50% advance received
3. Development starts
4. Dev server delivered (client-dev.get4domain.com)
5. Client review (5-7 business days)
6. Feedback incorporated
7. Final approval received in writing
8. 50% final payment received
9. Production deployment

## Rules

- Development starts ONLY after agreement + advance payment
- NO production deployment before full payment
- Dev server access revoked if payment delayed beyond 30 days
- Source code NOT included unless separately contracted (additional fee)

## Late Payment Policy

- 7 days overdue: reminder sent
- 15 days overdue: dev server suspended
- 30 days overdue: project on hold, restart fee applies
